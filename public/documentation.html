J-Flix Movie App Documentation

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL Response
Get List of Movies /movies GET
Return data about a movie by title /movies/by-title/:title GET
Return data about a director's Name /movies/by-director/:director-name GET
Return data about a director's Bio /movies/by-director/:director-bio GET
Return data about a director's Birth year /movies/by-director/:director-birth-year GET
Return data about a genre /movies/by-type/:genre-name GET
Return data about a genre description /movies/by-type/:genre-description GET
Return data about a movie image /movies/by-type/:image-path GET
Add a user /user POST
Update a user /user/by-id/update PUT
Add a movie to favorites /movies/add-to/favorites POST
Remove a movie from favorites /movies/delete-from/favorites DELETE
Allow user to deregister /user/by-id/delete DELETE

Description of Endpoints

Below you will find high-level description of what kind of data the endpoint returns, the endpoint URL, The method type (GET, POST, PUT, or DELETE), the query parameters (parameters in the URL), the format of data expected in the body of the request (and an example) and the format of the response data

GET /movies

Gets a list of movies from the array.

Endpoint URL

http://localhost:8080/movies
