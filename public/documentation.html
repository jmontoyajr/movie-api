J-Flix Movie App Documentation

Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL Response
Get List of Movies /movies GET
Return data about a movie by title /movies/:description/:genre/:director/:image GET
Return data about a director by Name /movies/:director/:bio/:birth-year/:death-year GET
Add a user /movies/[USER] POST
Update a user /movies/:username/:age/:phone PUT
Add a movie to favorites /movies/[FAVORITES] POST
Remove a movie from favorites /movies/[FAVORITES] DELETE
Allow user to deregister /movies/[USER] DELETE

Description of Endpoints

Below you will find high-level description of what kind of data the endpoint returns, the endpoint URL, The method type (GET, POST, PUT, or DELETE), the query parameters (parameters in the URL), the format of data expected in the body of the request (and an example) and the format of the response data

GET /movies

Gets a list of movies from the array.

Endpoint URL

http://localhost:8080/movies

GET /movies/:description/:genre/:director/:image

Gets a list of data based on the Movie title.

Endpoint URL

http://localhost:8080/movies/:description/:genre/:director/:image

GET /movies/:director/:bio/:birth-year/:death-year

Return data about a director by Name.

Endpoint URL

http://localhost:8080/movies/:director/:bio/:birth-year/:death-year

POST /movies/[USER]

Add a user.

Endpoint URL

http://localhost:8080/movies/[USER]

PUT /movies/:username/:age/:phone

Updates user information.

Endpoint URL

http://localhost:8080/movies/:username/:age/:phone

POST /movies/[FAVORITES]

Add a movie to favorites.

Endpoint URL

http://localhost:8080/movies/[FAVORITES]

DELETE /movies/[FAVORITES]

Remove a movie from favorites.

Endpoint URL

http://localhost:8080/movies/[FAVORITES]

Endpoint URL
